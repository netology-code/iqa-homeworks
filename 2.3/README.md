# Домашнее задание к занятию 2.3. «Жизненный цикл разработки ПО. Гибкие методологии разработки ПО»

Бабушка генерального директора нашего банка не очень хорошо разбирается в интернет-банкинге и постоянно там путается. Поэтому директор решил для всех пенсионных счетов сделать отдельную вкладку, в которой также будет осуществляться и перевод денег между счетами, и оплата коммнульных услуг. Так как видит бабушка не очень хорошо, то весь шрифт должен быть увеличен, а картинки более чёткими.

Рассмотрите кейс внедрения нового функционала в готовый продукт. 
Опишите, как можно контролировать качество данной фичи на всех этапах разработки и какие действия должны быть совершены тестировщиками (до разработки, во время, и уже после).
